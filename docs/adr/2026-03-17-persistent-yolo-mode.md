# ADR: 持久化 YOLO Mode

## 状态

已接受，生效日期 `2026-03-17`。

## 背景

V2 需要提供一个名为 `YOLO mode` 的高风险执行开关，使未来 Codex 线程在本地以：

- `approvalPolicy='never'`
- `sandboxMode='danger-full-access'`

运行。

争议点在于这个开关应当是“当前会话临时态”，还是“插件级持久化设置”。

## 决策

当前版本按产品要求，将 `YOLO mode` 设计为**持久化设置**。

实现约束：

1. 默认值为关闭。
2. 设置页必须明确提示它属于高风险模式。
3. 开启后，插件重载与 Obsidian 重启后仍保持开启。
4. 当用户手动把 `sandboxMode` 或 `approvalPolicy` 改为非 YOLO 组合时，`yoloMode` 自动关闭，避免状态错位。

## 取舍

### 选择持久化的收益

- 用户不需要在每次重启 Obsidian 后重新开启高权限执行模式。
- 设置语义更接近“默认执行策略”，而不是一次性会话按钮。

### 选择持久化的代价

- 风险持续时间更长，误配置影响范围更大。
- 用户可能忘记自己仍处于高风险模式。

## 风险缓解

1. UI 使用显式高风险描述。
2. 默认保持关闭。
3. 后续状态栏实现中使用红色警示态持续提示。
4. README 明确说明该设置会持久化。

## 后续影响

后续 Task 5 / Task 6 需要把 `yoloMode` 与状态栏、线程选项传递逻辑打通，但不改变本 ADR 的“持久化”结论。
