# Obsidian Codex V2 UI Refresh Design

## 背景

当前 V2 功能已经齐备，但界面仍然是“传统聊天面板 + 信息块堆叠”的风格，和参考图中的“克制空态 + 底部托盘”方向差距较大。问题不只是配色，而是信息层级、留白、操作区布局和卡片语言都不统一。

## 目标

将聊天侧边栏整体重构为更贴近参考图的单画布界面：

1. 顶部保留极简品牌头。
2. 中部默认是大面积留白的空态。
3. 底部输入区重构为一体化托盘，承载输入、轻量操作、模型选择、运行状态与 YOLO。
4. 消息卡片、状态栏、事件卡片统一为低噪声深色视觉语言。
5. 模型列表对齐用户提供的官方客户端截图。

## 核心设计

### 1. 整体骨架

- 页面分三层：顶部品牌条、中部内容区、底部托盘。
- 顶部只保留图标和标题，不再放主操作按钮。
- 中部在空会话时显示居中大标题；进入对话后平滑切换为消息流。
- 底部托盘固定在内容区底部，视觉上是一整块深色圆角容器。

### 2. 空态

- 使用 serif 大标题作为唯一视觉中心。
- 标题文案采用时间感问候语，风格接近参考图。
- 空态不堆叠辅助说明，不展示技术字段。

### 3. 托盘

- 托盘上部为轻量图标操作区。
- 中部是大输入框。
- 下部为状态带，左侧展示模型/推理强度/工作目录，右侧展示 YOLO 开关。
- 状态带并入托盘，不再单独显示一个“状态栏组件面板”。

### 4. 消息与事件卡片

- 取消明显彩色边框，统一改为柔和面板。
- 用户消息保留轻微强调，但不使用高饱和主色大面积铺底。
- `reasoning` 改为弱提示折叠面板。
- `command`、`file_change`、`activity` 统一使用次级文本和小标题，不再像独立组件库。
- 错误态依旧可辨识，但避免刺眼的纯红大片。

### 5. 模型与推理强度

- 模型列表直接对齐用户提供的客户端截图：
  - `GPT-5.4`
  - `GPT-5.3-Codex`
  - `GPT-5.2-Codex`
  - `GPT-5.2`
  - `GPT-5.1-Codex-Max`
  - `GPT-5.1-Codex-Mini`
- 底部托盘新增推理强度下拉，和模型选择并列。
- 显示名使用客户端文案，内部仍传递对应模型 ID / reasoning effort。

## 约束

1. 不破坏现有运行时逻辑、事件映射和设置持久化。
2. 保留 Obsidian 主题兼容性，优先复用 CSS 变量。
3. 自动化验证至少维持 `npm test`、`npm run typecheck`、`npm run build` 全绿。
4. 本轮重点是视觉与布局重构，不扩会话能力。
