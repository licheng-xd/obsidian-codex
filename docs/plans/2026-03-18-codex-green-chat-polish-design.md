# Codex Green Chat Polish Design

**背景**

当前 `feat(chat-ui): redesign codex chat shell` 已经建立了单画布 + 底部托盘结构，但视觉语言仍然偏暖色卡片化，和用户期望的 Claudian 风格仍有差距。新的目标是在保留 Codex 事件能力的前提下，把聊天界面进一步收敛成更像正文流的交互体验。

## 目标

1. 整体色系切换为 Codex CLI 风格的深色底 + 绿色强调色。
2. 底部托盘压缩成更轻、更小的控制条，YOLO 使用真正的左右滑动开关。
3. 托盘顶部上下文只显示 `Vault Root`，不再展示 `Current note` 和 `Selection`。
4. 普通对话尽量去卡片化，保留正文流观感；工具事件只保留弱层次块。
5. `Thinking` / 等待状态要有细节动画，并在输出完成后保留更小号的元信息提示。

## 视觉方向

- 画布保持深色、安静、大留白。
- 顶部品牌区保持极简，只作为定位锚点。
- 空态继续保留大标题，但进入对话后应快速退到背景层。
- 用户消息收敛成小型右侧气泡，不再是大面积色块。
- Assistant 消息改成主阅读流，强调排版和间距，而不是边框和背景。
- 所有状态、标签、辅助信息统一收缩为小号字体和低对比度层级。

## 结构调整

### 消息区

- 每轮 assistant 输出改成“回合块”：
  - 顶部是 `Thinking` 元信息。
  - 中间是正文流回复。
  - 底部按需插入工具事件块。
- `text` 事件不再渲染成明显卡片，而是正文排版块。
- `reasoning` 事件优先折算成轻量 meta 文案；仅在有具体 reasoning 文本时显示小号辅助段落。

### 系统事件流

- `reasoning`、`command`、`activity`、`file_change` 统一改造成“系统事件行”。
- 默认只显示一行摘要，不自动展开细节，视觉上参考 VS Code Codex 插件中的工具动作流。
- 摘要文案统一为动词式中文：
  - `已思考`
  - `已运行 <command>`
  - `已搜索 <query>` / `已调用 <tool>`
  - `已编辑的文件`
- 所有有明细的事件使用相同的轻量折叠结构；展开后再显示完整命令输出、文件列表、reasoning 文本或 activity detail。
- 普通 assistant 正文继续保持正文流，不被事件样式影响。

### 状态表达

- 发送后立即创建 `Thinking` 元信息占位。
- 等待阶段使用细微 shimmer / pulse 动画。
- 收到正文后动画停止，但 `Thought for <n>s` 保留。
- 颜色、字号、字体风格与正文明确区分。

### 底部托盘

- 顶部只保留 `Vault Root` 标签。
- 控制条压缩为一行，左侧是模型和推理强度，中部是 usage，右侧是 YOLO。
- YOLO 改为真正的滑块开关，激活时使用绿色高亮，而不是大面积警告色。
- 完整 Vault 路径放在 tooltip，默认只显示末级目录标签。

## 交互原则

- 普通消息默认不显示多余操作。
- 命令、文件变更、错误等非正文信息保留弱块，不抢占主视觉。
- 取消、发送、New Chat 图标按钮进一步缩小和降噪。
- 保持现有 Enter / Cmd+Enter 交互不变。

## 风险与约束

- 不能破坏现有流式渲染、取消和事件映射逻辑。
- 不能删除目前已经存在的模型 / 推理强度 / YOLO 功能，只能调整呈现。
- `turn.completed` usage 和工作目录信息仍需保留，只是要以更轻方式展示。
